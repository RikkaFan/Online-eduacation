# 学生端“我的课程”排序与考试按选课可见优化计划

## 1. 目标
- 将学生端左侧栏“我的课程”放到“考试列表”上方。
- 学生端课程与考试页面 UI 风格进一步对齐“学生端控制台首页”玻璃拟物化视觉。
- 实现“学生必须先选课，才能看到对应课程考试”的业务规则。

## 2. 涉及范围
- 前端：
  - `frontend/src/views/StudentView.vue`（菜单顺序）
  - `frontend/src/views/StudentExamList.vue`（考试列表样式与筛选逻辑）
  - `frontend/src/api/examTaking.js`（考试获取方式，如需新增接口）
- 后端：
  - `backend/src/main/java/com/example/onlineexam/controller/ExamController.java`
  - `backend/src/main/java/com/example/onlineexam/service/ExamService.java`
  - `backend/src/main/java/com/example/onlineexam/repository/ExamRepository.java`
  - `backend/src/main/java/com/example/onlineexam/repository/CourseEnrollmentRepository.java`（复用）

## 3. 实施步骤

### 步骤A：菜单顺序调整
- 在学生左侧栏中将“我的课程”菜单项移动到“考试列表”之前。
- 校验菜单激活状态与路由跳转行为不受影响。

### 步骤B：考试按“已选课程”过滤（核心）
- 后端新增/调整“学生考试列表”查询逻辑：
  - 基于当前登录学生ID读取其已选课程ID集合（`ENROLLED`）。
  - 仅返回这些课程下的考试数据。
- 方案优先：
  - 方案1：新增学生专用考试列表接口（推荐，语义清晰）。
  - 方案2：在现有课程考试接口里增加学生选课校验与过滤（改动较小但语义耦合）。
- 前端考试列表页改为调用“按已选课返回”接口，避免前端仅做展示过滤。

### 步骤C：UI风格对齐控制台首页
- 统一考试列表页视觉参数：
  - 容器玻璃背景、柔和大圆角、浅边框、分层阴影。
  - 标题区、筛选区、考试卡片区采用同一视觉层级。
  - 按钮、标签与状态色统一为当前学生端主风格。
- 同步微调“我的课程”入口所在侧栏项的视觉一致性（悬停/激活态保持统一）。

### 步骤D：空态与提示优化
- 学生未选课时，考试列表显示空态提示：
  - “请先到‘我的课程’选课后再参加考试”。
  - 提供快捷跳转按钮到“我的课程”。
- 学生已选课但暂无考试时，显示“已选课程暂无考试”空态文案。

### 步骤E：权限与边界校验
- 校验以下场景：
  - 学生退课后，课程对应考试不再可见。
  - 学生直接访问考试详情/准备页时，若未选课需阻断并提示。
  - 教师与管理员考试能力不受本次改动影响。

## 4. 验证方案
- 前端：
  - 学生登录后检查侧栏顺序正确。
  - 在“未选课/已选课”两种场景验证考试列表显示差异。
  - 验证空态提示与跳转按钮行为。
- 后端：
  - 验证学生考试查询仅返回已选课程数据。
  - 验证非学生角色逻辑不被误伤。
- 构建：
  - 前端 `npm run build` 通过。
  - 后端 `./mvnw -q -DskipTests compile` 通过。

## 5. 交付结果
- 菜单顺序调整完成（“我的课程”在“考试列表”上方）。
- 考试列表实现“按选课可见”。
- 学生端考试页UI风格与控制台首页一致性提升。
- 提供验证记录与关键改动说明。
