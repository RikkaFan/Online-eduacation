# 学生端考前与做题页视觉改版计划 v2

## 目标与约束
- 目标：优化“考前准备页 + 做题页”视觉与可读性，提升质感与专注度，让界面更接近你提供的参考图风格。
- 强约束：必须保持学生端控制台首页同款设计语言：
  - 纯粹玻璃拟物化
  - 极致柔和大圆角
  - 高通透呼吸感排版
  - 多层弥散光影
  - 局部微拟物交互
  - 克制极简图文层级
- 功能约束：不改变考试业务逻辑（计时、选题、作答、交卷流程不动），仅做结构与样式改造。
- 交互硬要求：选项必须靠左、竖向排列，接近你图中的“单列大行高 + 左侧圆形序号/选框”。

## 当前问题诊断（为什么“不好看”）

### 1) 视觉层级冲突
- 题干区、作答区、导航区、答题卡区都在抢主视觉，导致“阅读焦点”不稳定。
- 卡片边框、按钮色块、状态色同时偏强，缺少“主次递进”。

### 2) 空间比例不统一
- 左右栏宽度、内边距、模块间距没有形成统一基线，页面显得“散”和“扁”。
- 题干卡与选项区纵向节奏断裂，眼睛需要反复跳转。

### 3) 选项样式不符合考试阅读模型
- 目前选项交互偏组件默认样式，未形成“题目-选项”阅读连续性。
- 选项点击热区不够大，文字与控件关系弱，不像参考图那样“整行可读可点”。

### 4) 玻璃拟物风格一致性不足
- 当前存在“局部玻璃 + 局部系统默认”混用，导致风格断层。
- 弥散光影与圆角体系未形成统一 Token（半径、阴影、透明度、描边）。

## 改版方案（保持同款语言 + 对齐参考图）

### A. 全局布局重构（先做结构，再做皮肤）
- 将做题页拆分为三层主结构：
  1. 左侧信息侧栏（考试信息/倒计时/交卷）
  2. 中间主作答区（题干 + 选项 + 翻页）
  3. 右侧答题卡（题号矩阵 + 状态图例）
- 建立统一间距系统：外层、卡片内边距、模块间距全部采用一致步进（例如 8 的倍数体系）。
- 统一圆角系统（大卡、中卡、按钮、选项行）并保持呼吸留白。

### B. 题干区视觉降噪与聚焦
- 题干卡采用“高通透玻璃底 + 轻描边 + 柔阴影”。
- 题号/题型标签弱化，题干文本作为一级视觉主体（字号、行高、对比度优先）。
- 题干行宽控制在舒适阅读范围，避免过长一行影响理解。

### C. 选项区改成“左对齐竖向大行”
- 选项强制单列竖排，靠左对齐，逐行上下排列。
- 每个选项改为“整行可点”的 option-row（不是只有 radio/checkbox 可点）：
  - 左侧：圆形标识（A/B/C/D）或单选/多选态圆环
  - 右侧：选项文本
  - 状态：默认 / hover / 选中 / 键盘聚焦
- 行高与点击热区增大，提升触控与快速阅读体验。
- 判断题同样采用单列双行结构（正确/错误），保持统一。

### D. 交互与反馈微拟物化
- hover：轻微抬升与边框高亮，避免强烈动画。
- 选中：低饱和主色填充 + 内描边，强调“已选”但不刺眼。
- 当前题、已答、未答在答题卡中保持低噪状态色，保证信息可扫读。

### E. 考前准备页风格对齐
- 与做题页统一卡片比例、描边透明度、阴影层级和按钮风格。
- 关键信息（科目、时长、时间）做“主副层级”重排，减少视觉拥挤。
- 主按钮保持同款蓝色语义，但降低饱和冲突，增强高级感。

## 实施步骤（按顺序执行）

### 第1步：抽离视觉 Token
- 定义统一变量：圆角、阴影、边框透明度、背景透明度、间距、文本层级。
- 替换 ExamTaking 与 StudentExamReady 的硬编码样式，先统一基础风格。

### 第2步：重排 ExamTaking 结构
- 调整主容器网格比例和断点策略。
- 分离题干区与选项区容器，明确“题干先读、选项后答”的阅读路径。

### 第3步：重做选项组件样式与点击模型
- 将单选/多选/判断统一为 option-row 交互壳。
- 保证“整行点击”可选中；保持 v-model 逻辑不变。
- 强制竖向左对齐并统一行间距。

### 第4步：优化答题卡与底部操作区
- 答题卡题号矩阵按统一尺寸与间距重排。
- 当前题、已答、未答视觉差异保留但降噪。
- 上一题/下一题按钮尺寸、间距、阴影与首页风格一致。

### 第5步：同步改造 StudentExamReady
- 卡片比例、标题层级、描述区和主按钮改为同款玻璃拟物体系。
- 移动端保持同样层级逻辑，不做“堆样式”式压缩。

### 第6步：验证与回归
- 功能回归：单选/多选/判断/主观、切题、倒计时、交卷、答题卡跳转。
- 视觉回归：桌面端与移动端断点下无错位、无样式断层。
- 对照你的参考图确认“左对齐竖排选项”达到预期。

## 交付标准（完成判定）
- 选项区已实现左对齐单列竖排、整行可点。
- 做题页与考前页统一到学生端首页同款玻璃拟物语言。
- 页面主次清晰、阅读路径顺畅、视觉噪音显著下降。
- 所有考试核心功能行为与数据提交流程保持不变。
