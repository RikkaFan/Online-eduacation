# 学生端考试作答页 UI 重构计划（单题预览 + 整卷预览）

## 1. 目标与范围
- 页面范围：`frontend/src/views/ExamTaking.vue`
- 目标问题：
  - 中间栏选项框过大、选项间距过小，阅读拥挤
  - 题号与题干文字未良好对齐
  - “上一题/下一题”位置可下移到内容区底部
  - “整卷预览”布局需与单题预览保持左右栏一致，中间栏改为可滚动
  - 视觉风格对齐学生端控制台首页：玻璃拟物化、柔和大圆角、高通透、分层光影

## 2. 设计基线（复用现有风格）
- 参考学生端控制台首页风格来源：
  - `frontend/src/views/StudentDashboard.vue`
  - `frontend/src/assets/base.css`
  - `frontend/src/assets/main.css`
- 本次改造原则：
  - 保持既有信息结构与业务逻辑不变（答题、计时、提交、答题卡跳转）
  - 仅做布局与视觉层优化，避免改动判题/保存答案逻辑
  - 风格参数统一：更大圆角、轻透背景、细边框、双层柔和阴影、适度 hover 动效

## 3. 实施步骤

### 步骤 A：重构中间栏题目卡片排版（解决“拥挤/过大/不对齐”）
- 调整题目头部结构与样式：
  - 统一题号区域与题干主文本的基线对齐
  - 为题号与题干设置稳定的横向 gap，避免错位
- 调整选项容器密度：
  - 下调选项项高度与内边距（让选项框“稍微小一点”）
  - 增大相邻选项的垂直间距（缓解拥挤）
  - 优化选项字母圆圈与文本的对齐与间隔
- 保留并微调当前选中/未选中状态视觉对比，确保可读性不下降

### 步骤 B：调整单题预览布局重心（把翻题按钮下移）
- 将“上一题/下一题”从当前内容中部改到中间栏底部操作区
- 在中间栏增加“内容区 + 底部操作区”纵向分层：
  - 内容区优先展示题干与选项
  - 底部操作区固定或贴底显示翻题按钮
- 处理单题内容较短场景：底部保留呼吸留白，避免空白区域观感突兀

### 步骤 C：单独优化整卷预览布局（与单题预览同构）
- 保持左右栏（考试信息 + 答题卡）与单题模式一致，不再出现结构跳变
- 将中间栏设置为独立滚动容器：
  - 整卷题目列表在中间栏内滚动
  - 顶部工具区（模式切换）与左右栏保持稳定
- 优化整卷题块间距与卡片层次，避免“长列表挤压感”

### 步骤 D：统一玻璃拟物化视觉语言（局部升级）
- 在 `ExamTaking.vue` 中对核心容器（左栏卡片、中间题卡、右栏答题卡）统一升级：
  - 更柔和大圆角
  - 半透明底 + backdrop blur
  - 细描边 + 分层阴影
- 统一按钮、标签、状态块的圆角与光影参数，保证同页视觉一致性
- 保持可访问性：文本对比度、交互状态可辨识度不降低

### 步骤 E：响应式与滚动细节校准
- 校准桌面主分辨率和中等宽度断点（现有 1280 断点基础上优化）
- 确保中间栏滚动时左右栏不抖动，滚动条样式与玻璃风格一致
- 检查超长题干/超多选项/大题型文本的换行与间距稳定性
