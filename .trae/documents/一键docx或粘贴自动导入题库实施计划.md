# 一键docx或粘贴自动导入题库实施计划

## 结论（直接回答你的问题）
- 可以，而且应该做成一个按钮：**“一键解析并导入”**。  
- 我之前拆成“先转Excel再导入”是为了最低风险上线；你现在这个诉求是更好的产品体验，完全可行。

## 目标
- 支持两种输入：
  - 上传 `.docx`
  - 粘贴题库文本
- 点击一个按钮后自动完成：
  1. 解析题目
  2. 数据校验
  3. 调用现有批量导入接口
  4. 返回成功/失败明细
- 用户不再手动导出 Excel、再二次上传。

## 总体方案
- 前端新增“智能导入”入口（在题库管理页导入区域）。
- 解析阶段可在前端完成（文本规则解析），提交阶段复用现有后端导入逻辑。
- 为保证稳定性，保留“预览+确认导入”开关（默认开启），但支持“跳过预览直接导入”。

## 详细实施步骤

### 第1步：统一输入管道
- 新增导入面板，提供：
  - `docx上传`
  - `文本粘贴`
  - `课程选择`
  - `一键解析并导入` 按钮
- `docx` 读取为纯文本，和粘贴文本走同一解析器，避免两套逻辑。

### 第2步：实现题目解析器（覆盖常见教师题库格式）
- 题目切分：支持 `1.`、`1、` 等题号。
- 选项识别：支持 `A.` / `A、` / `A ` 多写法。
- 答案识别：支持 `【答案】B`、`答案：B`、`参考答案：B`。
- 解析识别：支持 `解析：...`、`【解析】...`。
- 题型推断：
  - 单答案字母 -> `SINGLE`
  - 多答案（如 `A,C`）-> `MULTIPLE`
  - 判断关键词 -> `JUDGE`
  - 无选项长文本可落 `SUBJECTIVE`

### 第3步：一键导入执行链路
- 点击按钮后执行：
  1. 解析文本为结构化题目数组
  2. 本地校验（题干、答案、课程ID必填）
  3. 异常题目收集（不合法项）
  4. 生成后端可接收的批量数据
  5. 调用导入接口写入数据库
  6. 回显结果（成功数、失败数、失败原因）
- 导入成功后自动刷新题库列表。

### 第4步：错误处理与回退策略
- 若“可解析题目=0”，不调用导入接口，直接提示格式问题。
- 若部分题目失败，展示失败清单并允许“仅重试失败项”。
- 若一键导入失败，提供“导出为Excel模板”作为兜底路径（保证可用性）。

### 第5步：后端最小改动（仅必要）
- 保持现有导入接口不破坏兼容。
- 增加一个轻量批量JSON导入端点（可选）以免前端再绕Excel；若不新增端点则复用现有上传流程。
- 所有导入写操作继续执行教师归属校验（只能导入到本人课程）。

### 第6步：权限与安全
- 仅 `TEACHER/ADMIN` 可用该功能。
- 教师导入时强制校验 `courseId` 归属当前教师。
- 限制单次导入题量与文本大小，防止误操作和性能抖动。

### 第7步：验收标准
- 能直接处理你给的样例格式并成功入库。
- 一键流程中无需下载/上传中间Excel文件。
- 对失败题目可定位到具体题号和原因。
- 教师A导入的数据教师B不可见（沿用现有隔离规则）。

## 为什么“一键”比“先转Excel”更合适
- 少一步人工操作，教师更容易用。
- 减少模板填错、二次上传失败的概率。
- 依然可保留Excel兜底，不牺牲稳定性。

## 分阶段上线建议
- Phase 1：先做“一键解析 + 预览 + 导入”（主流程可用）
- Phase 2：补“跳过预览直接导入”“失败项重试”
- Phase 3：增强复杂文档解析与题型识别准确率
