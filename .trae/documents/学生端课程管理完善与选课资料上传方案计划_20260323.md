# 学生端课程管理完善与“选课 + 资料上传”实施计划

## 一、现状确认
- 学生端课程页已存在：`/student/courses` 对应 `StudentCourseList.vue`，页面已含课程进度与评价能力。
- 当前核心问题是“入口可达性不足”：学生左侧栏未挂“我的课程”菜单，导致看起来像未实现。
- 选课能力与课程资料上传能力目前在代码中尚未形成完整业务闭环（无选课关系模型、无课程附件字段与文件流程）。

## 二、目标定义
- 目标1：学生端左侧栏新增“我的课程”入口，直达课程管理页。
- 目标2：在不破坏现有课程学习/进度/评价功能的前提下，补齐选课机制。
- 目标3：给出课程资料上传的可落地方案，并与教师端课程管理联动。
- 目标4：明确论文中“当前已实现”和“本次新增实现”的边界，避免答辩口径冲突。

## 三、实施范围
- 前端：
  - `frontend/src/views/StudentView.vue`
  - `frontend/src/views/StudentCourseList.vue`
  - 课程相关 API 文件（新增选课与资料接口）
- 后端：
  - 课程域新增“选课关系”实体与接口
  - 章节域新增“资料文件元数据”字段/表与上传下载接口
  - 权限控制（学生仅能管理本人选课，教师仅能管理本人课程资料）

## 四、实施步骤

### 步骤1：先做可见性修复（低风险快速收益）
- 在学生左侧栏补“我的课程”菜单项，路由跳转到 `/student/courses`。
- 校验当前菜单高亮、路由守卫、学生角色权限链路。
- 验证：学生登录后可一键进入课程管理页，页面已有进度与评价可正常使用。

### 步骤2：选课能力设计与实现
- 数据模型：
  - 新增 `course_enrollments`（建议字段：`id/course_id/student_id/status/create_time/update_time`）。
  - 增加唯一约束 `(course_id, student_id)`，避免重复选课。
- 后端接口（建议）：
  - `POST /api/courses/{courseId}/enroll`（学生选课）
  - `DELETE /api/courses/{courseId}/enroll`（学生退课）
  - `GET /api/courses/enrolled/me`（学生我的课程）
  - `GET /api/courses/{courseId}/enrollments`（教师查看选课名单，可选）
- 前端交互：
  - 课程列表新增“选课/已选/退课”状态按钮。
  - “我的课程”视图默认显示已选课程，并保留筛选切换“全部/已选”。
- 权限规则：
  - 学生只能操作本人选课；
  - 教师只能查看本人课程的选课数据；
  - 管理员可审计但不强制参与教学操作。

### 步骤3：课程资料上传方案落地（教师上传，学生查看）
- 资料上传建议采用“教师侧上传、学生侧只读下载”模式（教学场景最稳妥）。
- 数据结构建议：
  - 方案A（推荐）：新增 `chapter_materials` 表（支持多文件）
  - 方案B（简化）：在 `chapters` 增附件URL字段（仅单文件）
- 接口建议：
  - `POST /api/chapters/{id}/materials`（教师上传）
  - `GET /api/chapters/{id}/materials`（教师/学生查看）
  - `DELETE /api/chapters/{id}/materials/{materialId}`（教师删除）
- 存储策略建议：
  - 本地开发阶段：本地文件目录 + 静态资源映射；
  - 可扩展：预留对象存储适配层（后续切 OSS/MinIO）。
- 前端展示：
  - 教师课程管理页增加“上传资料”入口；
  - 学生课程学习抽屉中增加“资料下载区”。

### 步骤4：联调与边界校验
- 关键联调场景：
  - 未选课学生是否允许看课程资料（建议：仅已选可见）
  - 退课后进度与评价数据保留策略（建议：保留历史、隐藏入口）
  - 课程删除时选课与资料级联策略
- 安全校验：
  - 上传文件类型白名单、大小限制、文件名规范化
  - 下载鉴权与路径穿越防护
  - 接口限权与越权访问测试

### 步骤5：论文同步更新
- 第二章需求分析补“选课管理、课程资料管理”。
- 第三章数据库设计补 `course_enrollments`、`chapter_materials`（若采用方案A）。
- 第四章详细实现补“选课流程时序、资料上传下载流程”。
- 第五章测试补：
  - 选课/退课功能测试
  - 资料上传下载权限测试
  - 文件上传安全测试

## 五、方案选择建议（直接给结论）
- 选课：必须新增独立关系表，不建议“课程表加学生字段”。
- 资料上传：推荐“章节多附件表 + 教师上传学生只读下载”，可维护性最好。
- 页面策略：先补左侧栏入口，再做选课和资料上传，风险最低、收益最大。

## 六、验收标准
- 学生端左侧栏可直达“我的课程”。
- 学生可完成选课/退课，“我的课程”只展示本人已选课程。
- 教师可为章节上传资料，学生可在课程学习页查看/下载资料。
- 关键权限与异常场景可复现通过（越权访问、重复选课、非法文件上传）。
- 论文对应章节可与代码实现一一映射。

## 七、实施顺序与里程碑
- 里程碑M1：左侧栏入口补齐 + 现有课程页可达性验证
- 里程碑M2：选课模型与接口落地 + 前端选课交互完成
- 里程碑M3：资料上传下载链路落地 + 权限与安全校验完成
- 里程碑M4：测试证据与论文章节同步完成
